# 🔐 SSH Brute Force 공격 탐지 및 로그 분석 프로젝트

## 📌 프로젝트 개요
본 프로젝트는 SSH 서비스에 대한 Brute Force 공격을 수행하고, 발생한 로그를 분석하여 공격 패턴을 식별하는 것을 목표로 한다.

---

## 🖥️ 환경 구성
- 공격자: Kali Linux (192.168.85.134)
- 피해자: Rocky Linux (192.168.85.130)
- 네트워크: NAT 환경

---

## 💣 공격 수행
Hydra를 사용하여 SSH brute force 공격 수행

```bash
hydra -l root -P pass.txt ssh://192.168.85.130 -t 4

### 🔎 분석 결과
- 동일 IP에서 다수 로그인 실패 발생
- 짧은 시간 내 반복 시도
- 자동화 공격 도구 사용 의심

👉 Brute Force 공격으로 판단

---

## 📊 탐지 기준
- 동일 IP에서 짧은 시간 내 5회 이상 로그인 실패 발생 시 공격으로 판단

---

## 🛡️ 대응 방안
- Fail2Ban을 통한 IP 차단
- root SSH 접속 제한
- SSH 보안 설정 강화

---

## 🧠 느낀점
로그 분석을 통해 단순한 이벤트가 아닌 공격 흐름을 파악하는 것이 중요하다는 것을 알게 되었다. 실제 보안관제 업무에서 로그 분석 능력이 핵심임을 확인하였다.
